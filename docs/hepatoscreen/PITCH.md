## Слайд 2 — Проблема

### На слайде:
- **Заголовок:** 1.6 млрд человек с NAFLD в мире. В Казахстане — нет системы скрининга.

| Факт | Источник |
|------|----------|
| Глобальная распространённость NAFLD: **32.4%** взрослого населения | Riazi et al., Hepatology 2023 (PMC10026948) |
| HBV среди общего населения Казахстана: **3.4–5.5%** | Kruglov 1995; Nurgalieva 2007 (PMC11094609) |
| HCV среди ПИН в Казахстане: **до 90.2%** | Zhussupov et al. 2007 (PMC11094609) |
| HBV среди ПИН в Казахстане: **до 79.7%** | Zhussupov et al. 2007 (PMC11094609) |

- **Gap в ПМСП:**
  - ❌ Нет систематического скрининга ХЗП и ХВГ
  - ❌ FIB-4/APRI считаются вручную — или не считаются вообще
  - ❌ УЗИ-пакеты не структурированы, нет AI-интерпретации
  - ❌ Нет триажа: кого срочно к гепатологу, кого оставить в ПМСП
  - ❌ Сёла и удалённые районы — гепатолога нет физически

- **Экономика:** Поздняя диагностика цирроза = **$15,000–50,000/год** на пациента (трансплантация, TIPS, госпитализации). Ранний скрининг — **< $50/пациент**.

### Текст спикера:
"В мире 1.6 миллиарда человек живут с NAFLD — неалкогольной жировой болезнью печени. В Казахстане распространённость вирусного гепатита B в общем населении достигает 5.5%, а среди уязвимых групп — почти 80%. Но в первичном звене нет системы скрининга. Врач ПМСП видит пациента 15 минут, не знает, считать ли ему FIB-4, не понимает, что делать с УЗИ, и точно не знает — отправлять к гепатологу или нет. А в селе гепатолога просто нет. Поздняя диагностика цирроза обходится системе в десятки тысяч долларов на пациента. Мы меняем это."

### Speaker notes:
- **Ключевое сообщение:** не "у нас крутой AI", а "система здравоохранения теряет людей и деньги из-за отсутствия триажа"
- Цифры — из peer-reviewed источников (PMC), жюри это ценит
- Упомяните село — это эмоциональный hook, показывает социальную значимость
- Если спросят про экономику — уточните: цифры усреднённые по literature, локальная казахстанская оценка требует пилотных данных
- **Возможное возражение:** "В МЗ уже есть программы гепатита" — ответ: "Есть программы лечения, но нет системы скрининга и триажа на уровне ПМСП"

## Слайд 3 — Решение

### На слайде:
- **Заголовок:** 3 контура. Fusion scoring. Explainable AI. Один docker compose up.

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  Clinical       │  │   ML Lab        │  │   ML API        │
│  Platform       │  │   (тест модели) │  │   (inference)   │
│  :3004          │  │   :3005         │  │   :8000         │
│                 │  │                 │  │                 │
│ • Регистр       │  │ • Загрузка УЗИ  │  │ • FIB-4 / APRI  │
│ • Кейсы         │  │ • Inference     │  │ • Fusion score  │
│ • Экспорт       │  │ • Grad-CAM      │  │ • Vision model  │
│   датасета      │  │ • Reasoning     │  │ • Explainability│
└────────┬────────┘  └────────┬────────┘  └────────┬────────┘
         │                    │                    │
         └────────────────────┼────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │   Go API :8088    │
                    │  Auth, DB, Cases  │
                    │   PostgreSQL      │
                    └───────────────────┘
```

- **Fusion scoring:** Клинические маркеры (FIB-4, APRI) + УЗИ-признаки → единый triage score
- **Explainability:** Grad-CAM overlay на УЗИ + reasoning popup — врач видит, почему ИИ принял решение
- **Deployment:** Docker Compose, pnpm monorepo, деплой за 5 минут

### Текст спикера:
"HepatoScreen — это не калькулятор FIB-4. Это три контура, работающие как единая экосистема. Клиническая платформа — регистр пациентов, кейсы, экспорт датасетов для МЗ. ML Lab — песочница, где врач загружает УЗИ, видит inference модели и explainability. ML API — сердце системы: FIB-4, APRI, fusion scoring, который объединяет клинику и УЗИ, и explainable AI — Grad-CAM overlay плюс reasoning, чтобы врач доверял решению. Всё поднимается одной командой docker compose up — и работает на ноутбуке с RTX 5050."

### Speaker notes:
- **Дифференциация:** Этот слайд — ключевой. Не просто калькулятор, не просто УЗИ-анализатор — а fusion + explainability + clinical workflow
- Акцент на docker compose up — показывает production-readiness даже на уровне прототипа
- Если спросят про архитектуру — упомяните: Go API отвечает за auth и персистентность, FastAPI — за ML inference, Next.js — за UI
- **RTX 5050** — упомяните как proof of concept, но скажите, что для production нужен GPU-кластер (заведите на Ask-слайд)

## Приложение — Ответы на сложные вопросы (FAQ для спикера)
